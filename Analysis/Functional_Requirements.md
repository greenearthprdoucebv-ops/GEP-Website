# Functional Requirements 

**Project:** Green Earth Produce Website  
**Author:** Preslav Stoyanov  
**Date:** March 2026

This document lists the main **functional** things the website should be able to do.  
(Non-functional requirements like performance, responsiveness, security, etc. are not included here.)

---

## Users

- **Visitor / Customer**: someone who visits the website to learn about the company and products.
- **Administrator**: someone from the company who can update the content.

---

## Functional Requirements

### Public website (Visitor)

- **FR1 Homepage**: Show a homepage that introduces Green Earth Produce and what they do.
- **FR2 Navigation**: Provide a clear menu to access the main pages.
- **FR3 Company info**: Show information about the company (about, services, location, contact details).
- **FR4 Products overview**: Show the products that Green Earth Produce offers.
- **FR5 Product details**: Allow the user to open a product and see more details.
- **FR6 Contact form**: Allow visitors to send a message/question using a contact form.
- **FR7 Confirmation**: After a contact form submission, show a “message sent” confirmation.

### Admin (Content updates)

- **FR8 Update content**: Admin can update company information and product content (text/images).

---

## Short user stories 

- As a visitor, I want to quickly understand what the company does, so I know if it fits my needs.
- As a visitor, I want to view products, so I can see what the company offers.
- As a visitor, I want to contact the company, so I can ask for pricing or cooperation.
- As an admin, I want to update content, so the website stays correct and up to date.
